🚗 FleetBox — Proxmox installation

Step-by-step guide: from your repository to a running LXC container.

Debian 12 LXC systemd service SQLite unprivileged
Before you start: the installer fetches the code with git clone, so FleetBox must live in a reachable Git repository first. Push it to GitHub (Step 1) or point REPO_URL at your own repo.

Prerequisites

  • Proxmox VE 7 or 8 with shell/root access to the host (not a guest)
  • A template storage (default local) and disk storage (default local-lvm)
  • Internet access from the host (template + package download)
  • FleetBox available in a reachable Git repository
1

Push the code to GitHub

On your workstation, in the project folder:

# in C:\Users\johan\claude\fleetbox
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/skyhell/fleetbox.git
git push -u origin main

The repository URLs already point to skyhell. If you forked this project, replace skyhell with your own GitHub user in README.md and proxmox/fleetbox.sh (the REPO_URL variable).

No GitHub? Skip this step and pass your own REPO_URL=... in Step 3, or use the manual copy approach.
2

Open a shell on the Proxmox host

Use SSH, or the web UI: Datacenter → your node → Shell. You are now root on the host. Verify:

pct list        # only exists on the Proxmox host
3

Run the installer

Defaults

bash -c "$(curl -fsSL https://raw.githubusercontent.com/skyhell/fleetbox/main/proxmox/fleetbox.sh)"

With custom values

CTID=151 HOSTNAME=garage DISK_GB=6 RAM_MB=1024 CORES=2 \
REPO_URL=https://github.com/skyhell/fleetbox.git \
bash -c "$(curl -fsSL https://raw.githubusercontent.com/skyhell/fleetbox/main/proxmox/fleetbox.sh)"

What the installer does

  1. Picks a free container ID (pvesh get /cluster/nextid).
  2. Downloads the Debian 12 LXC template if needed.
  3. Creates an unprivileged LXC (DHCP, nesting=1, onboot=1).
  4. Installs git, python3, venv.
  5. Clones FleetBox into /opt/fleetbox and builds a virtualenv.
  6. Writes .env with a random FLEETBOX_SECRET_KEY and a SQLite DB under /opt/fleetbox/data/.
  7. Initializes the database (app.cli init-db).
  8. Installs and starts the fleetbox systemd service.
  9. Prints the container IP and URL.
4

First-run setup in the browser

  1. Open the printed address, e.g. http://192.168.x.y:8000.
  2. Register — the first account automatically becomes the administrator.
  3. Top-right → your username → Account security → Enable 2FA (scan the QR code).
  4. Create more users from the admin Users page.
Disable self-registration once your users exist:
pct exec <CTID> -- sed -i 's/FLEETBOX_ALLOW_REGISTRATION=true/FLEETBOX_ALLOW_REGISTRATION=false/' /opt/fleetbox/.env
pct exec <CTID> -- systemctl restart fleetbox

Operation & maintenance

# Status / logs
pct exec <CTID> -- systemctl status fleetbox
pct exec <CTID> -- journalctl -u fleetbox -f

# Restart
pct exec <CTID> -- systemctl restart fleetbox

# Update to a new version
pct exec <CTID> -- bash -c "cd /opt/fleetbox && git pull && .venv/bin/pip install -r requirements.txt && systemctl restart fleetbox"

# 2FA recovery if a user lost their authenticator
pct exec <CTID> -- /opt/fleetbox/.venv/bin/python -m app.cli disable-2fa --username alice
Backup: it is enough to back up the SQLite file /opt/fleetbox/data/fleetbox.db inside the container.

Troubleshooting

SymptomCause / fix
pct: command not foundYou are not on the host. Switch to the Proxmox host.
Template … not foundRun pveam update; optionally set TEMPLATE_STORAGE=local.
git clone fails / 404Wrong REPO_URL or private repo. Fix the URL or make it public.
Page not reachableCheck IP with pct exec <CTID> -- hostname -I and read the logs.
Want HTTPSPut Caddy/nginx/Traefik in front — see docs/security.md.

Installing without GitHub

Don't want to publish to a Git repository? Use the local installer — it packages the source on the host and pushes it into the container. No Git, no internet repository required.

Option A — automated (proxmox/fleetbox-local.sh)

  1. Copy the project folder onto the Proxmox host:
# from your workstation
scp -r ./fleetbox root@PROXMOX-HOST:/root/fleetbox
  1. On the host, run the local installer from inside that folder:
cd /root/fleetbox
bash proxmox/fleetbox-local.sh

The same CTID, HOSTNAME, DISK_GB, … variables apply. The script creates the LXC, copies the code to /opt/fleetbox, and runs scripts/install.sh inside the container.

Option B — fully manual

Create a container yourself, copy the source into /opt/fleetbox (scp / pct push), then run the shared provisioning script:

pct exec <CTID> -- bash /opt/fleetbox/scripts/install.sh
scripts/install.sh also works on any plain Debian/Ubuntu host (no Proxmox): copy the project to /opt/fleetbox and run sudo bash scripts/install.sh.
