# Internationalization (i18n)

FleetBox ships with **German (`de`)** and **English (`en`)** translations. The
UI is translated; the documentation and source code are English-only.

## How it works

Translations are flat JSON files in `app/locales/<locale>.json`:

```json
{
  "vehicle.add": "Add vehicle",
  "service.type.oil_change": "Oil change"
}
```

Templates use the injected `t()` helper:

```html
<a href="/vehicles/new">{{ t('vehicle.add') }}</a>
```

`t()` formats `{placeholder}` arguments and falls back to the default locale,
then to the key itself if a string is missing.

## Locale resolution order

For each request the active locale is chosen from the first available signal:

1. `?lang=de` / `?lang=en` query parameter (also stored in the session)
2. The session value from a previous request
3. The logged-in user's `locale` preference
4. The browser `Accept-Language` header
5. `FLEETBOX_DEFAULT_LOCALE` (default `de`)

The language switcher in the top bar simply links to `?lang=<code>`.

## Adding a new language

1. Copy `app/locales/en.json` to `app/locales/<code>.json` and translate the
   values.
2. Add the code to `supported_locales` in `app/config.py`.
3. Restart the app. The switcher picks it up automatically.

## Adding a new string

1. Add the key to **both** `de.json` and `en.json`.
2. Reference it via `{{ t('your.key') }}` in templates.

Keep keys namespaced by area (`vehicle.*`, `service.*`, `fuel.*`, `stats.*`,
`search.*`, `attachment.*`, `backup.*`, `admin.*`, `nav.*`, `theme.*`,
`common.*`).
